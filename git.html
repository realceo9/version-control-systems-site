Git — основы и возможности

Git — это распределённая система контроля версий, позволяющая отслеживать изменения в коде, работать в команде и управлять ветками проекта.

Основные команды Git:

  • git init — создаёт новый локальный репозиторий в текущей папке. После этой команды в каталоге появляется скрытая папка .git, где хранится история проекта.
  • git add <файл> — добавляет указанный файл в индекс (в "список на коммит"). Чтобы добавить все файлы, можно использовать git add ..
  • git status — показывает состояние репозитория: какие файлы изменены, добавлены или не отслеживаются.
  • git commit -m "сообщение" — сохраняет добавленные изменения в истории репозитория с комментарием.
  • git log — выводит историю всех коммитов с авторами и датами.
  • git branch — показывает список веток, а также позволяет создавать их git branch имя_ветки.
  • git checkout имя_ветки — переключает текущую ветку. Чтобы создать новую ветку и сразу на неё перейти, можно использовать git checkout -b имя_ветки.
  • git merge имя_ветки — объединяет указанную ветку с текущей, сливая изменения.
  • git remote add origin <ссылка> — связывает локальный репозиторий с удалённым (например, на GitHub).
  • git push origin main — отправляет коммиты из локальной ветки main в удалённый репозиторий.
  • git pull origin main — получает и объединяет изменения с удалённого репозитория в текущую ветку.

Преимущества Git:

  • Распределённая система контроля версий — каждый разработчик имеет полную копию репозитория, что повышает надёжность и автономность.
  • Мощная работа с ветками — позволяет создавать отдельные ветки для экспериментов, новых функций и исправлений без риска сломать основной проект.
  • История изменений доступна локально — можно просматривать старые версии файлов и откатывать изменения без подключения к серверу.
  • Широкое сообщество и интеграции с GitHub — доступно множество ресурсов, плагинов и готовых решений для совместной работы и автоматизации.
  • Эффективная работа с командами — Git облегчает совместную разработку, объединение изменений и разрешение конфликтов.

Недостатки Git:

  • Крутая кривая обучения — новичкам сложно разобраться с концепциями ветвления, коммитов, слияния и разрешения конфликтов.
  • Избыточность возможностей — множество команд и опций может запутать тех, кто работает с небольшими проектами.
  • Неудобство при централизованной модели — в командах, где все изменения проходят через один сервер, Git может казаться избыточным.
  • Проблемы с большими бинарными файлами — Git плохо справляется с файлами, которые часто изменяются и не могут быть эффективно дифференцированы.

Типичный рабочий процесс в Git

Работа с Git обычно строится по повторяющемуся циклу действий. Этот процесс помогает отслеживать изменения, работать в команде и синхронизировать проект с удалённым репозиторием.

  1. Клонирование проекта. Если проект уже существует на GitHub или другом сервере, его копируют командой git clone <ссылка-на-репозиторий> (после этого появляется локальная копия репозитория со всей историей изменений).
  2. Создание новой ветки. Перед началом работы создаётся отдельная ветка командой git checkout -b имя_ветки (это помогает изолировать изменения от основной версии проекта).
  3. Редактирование файлов. Разработчик изменяет, добавляет или удаляет файлы в рабочей директории (эти изменения пока не сохранены в истории Git).
  4. Добавление изменений. После редактирования файлы добавляют в индекс командой git add . (это подготавливает их к фиксации).
  5. Коммит изменений. Сохраняют изменения в историю командой git commit -m "описание изменений" (каждый коммит содержит дату, автора и сообщение).
  6. Отправка на сервер. Чтобы обновления появились на GitHub, выполняют git push origin main (или другую ветку, если работа ведётся не в main).
  7. Получение обновлений. Для синхронизации с последними изменениями от других разработчиков используют git pull (эта команда скачивает и объединяет новые коммиты).
