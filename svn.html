SVN — основы и возможности

Subversion (SVN) — централизованная система контроля версий, в которой основная копия репозитория хранится на сервере. Разработчики получают рабочие копии, вносят изменения и отправляют их обратно на сервер. Такой подход удобен для проектов с централизованным управлением доступом и единой точкой контроля.

Основные команды SVN

  • svn checkout <URL> — загружает (клонирует) репозиторий с сервера на локальный компьютер.
    Дополнительно: svn info покажет сведения о текущем репозитории.
  • svn update — обновляет вашу локальную копию до последней версии с сервера, включая изменения от других участников.
    Совет: используйте перед началом работы, чтобы избежать конфликтов.
  • svn add <файл> — добавляет новые файлы под контроль версий, чтобы их можно было отправить на сервер.
    Сопутствующая команда: svn delete <файл> — удаляет файл из репозитория.
  • svn commit -m "сообщение" — сохраняет изменения и отправляет их на сервер.
    Рекомендуется писать короткое, понятное сообщение, описывающее внесённые правки.
  • svn status — показывает список изменённых, добавленных и неотслеживаемых файлов.
    Полезно перед коммитом, чтобы убедиться, что вы не пропустили нужные файлы.
  • svn log — выводит историю изменений проекта с указанием авторов, дат и комментариев.
    Можно добавить -r для просмотра конкретных ревизий, например svn log -r 10:20.
