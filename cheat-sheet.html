Git — Cheat Sheet

Краткий набор команд и паттернов, которые чаще всего используют в работе с Git.

Начало

  • git init — инициализировать репозиторий.
  • git clone <url> — склонировать удалённый репозиторий.
  • git status — текущее состояние.
  • git add . — добавить все изменения в индекс.
  • git commit -m "message" — зафиксировать изменения.

Ветки

  • git branch — показать ветки.
  • git branch <name> — создать ветку.
  • git checkout <name> — переключиться на ветку.
  • git checkout -b <name> — создать и переключиться.
  • git merge <branch> — слить в текущую ветку.
  • git rebase <base> — перенести коммиты на другую базу.

Синхронизация

  • git remote -v — посмотреть remotes.
  • git remote add origin <url> — добавить удалённый.
  • git push origin <branch> — отправить ветку.
  • git pull --rebase — подтянуть и применить поверх (рекоммендовано).
  • git fetch — получить обновления без слияния.

История и откат

  • git log --oneline --graph — компактный лог.
  • git diff — различия незакоммиченных изменений.
  • git checkout -- <file> — откатить файл к последнему коммиту.
  • git revert <commit> — отмена коммита новым коммитом.
  • git reset --soft <commit> — откат ветки, сохранив staged.

Продвинуто

  • git stash — временно убрать незакоммиченные изменения.
  • git stash pop — вернуть сохранённые изменения.
  • git cherry-pick <commit> — применить отдельный коммит в текущую ветку.
  • git bisect — бинарный поиск регрессии.
