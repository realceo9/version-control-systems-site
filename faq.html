Troubleshooting & FAQ

Короткие решения типичных проблем при работе с Git и SVN.

Merge conflict

Симптом: при merge или pull возникают конфликты.

Решение (шаги):

  1. Откройте конфликтующие файлы и найдите маркеры.
  2. Разрешите изменения вручную, сохраните файл.
  3. git add <file> для каждого разрешённого файла.
  4. git commit чтобы завершить merge.
  5. Если нужно откатиться: git merge --abort (если merge не закоммичен).

Detached HEAD

Симптом: вы находитесь не на ветке, а на конкретном коммите (HEAD detached).

Решение:

  • Создайте ветку из текущего состояния: git checkout -b fix-branch.
  • Затем можно пушить новую ветку и продолжать работу.

Ошибка при push (rejected)

Симптом: remote rejected push — non-fast-forward.

Решение:

  1. git pull --rebase чтобы подтянуть изменения и выполнить rebase ваших коммитов.
  2. Разрешите конфликты, затем git push.
  3. Если ситуация критична и вы уверены: git push --force-with-lease (использовать осторожно).

Большие файлы

Git плохо справляется с большими бинарными файлами. Решение — использовать Git LFS.

SSH vs HTTPS

Если push требует логин/пароль — можно переключиться на SSH:


            ssh-keygen -t ed25519 -C "youremail@example.com"
            # затем добавьте публичный ключ в GitHub (Settings → SSH and GPG keys)

После этого используйте remote с SSH-URL: git remote set-url origin git@github.com:USERNAME/REPO-NAME.git

Если ничего не допомогло

Сделайте копию рабочей папки, создайте issue в репозитории с описанием шагов и выводом команд git status и git remote -v — это поможет быстрее разобраться.
